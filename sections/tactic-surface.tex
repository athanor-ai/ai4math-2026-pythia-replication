\section{The pythia tactic surface}
\label{sec:tactic-surface}

\texttt{pythia} ships eleven user-facing tactics, two named aesop
rulesets, and six registries (five attribute-driven plus the
per-domain \texttt{DomainCalculator} typeclass). The design follows
the CoqHammer pattern \citep{czajkakaliszyk2018coqhammer}: a
domain-specific dispatcher with extensible attribute-driven lemma
selection, a fall-through to general-purpose automation, and
kernel-checked reconstruction for any external-oracle steps. The
headline tactic \texttt{pythia!} orchestrates the full
deterministic ladder under one call. The other tactics are
exposed for direct invocation when the user knows the goal shape.

\paragraph{The nine-rung ladder of \texttt{pythia!}.}
The headline tactic walks the rungs in order and returns the first
closing proof.
\begin{enumerate}[leftmargin=1.5em,itemsep=0.1em]
\item \texttt{stat\_simp} (curated probability and ENNReal normal
  forms).
\item \texttt{linarith} / \texttt{nlinarith} / \texttt{polyrith}
  (arithmetic).
\item \texttt{positivity}.
\item \texttt{aesop} on the registered \texttt{Pythia} ruleset.
\item The \texttt{@[stat\_lemma]} shape-dispatch cascade.
\item \texttt{z3\_check} (QF\_LRA via Z3).
\item \texttt{cvc5\_check} (QF\_BV / QF\_LRA via CVC5).
\item \texttt{vampire\_check} / \texttt{e\_check} (first-order logic
  via Vampire / E).
\item \texttt{disprove} (counterexample finder via Z3 model
  extraction).
\end{enumerate}

\paragraph{\texttt{pythia} (top-level dispatcher).}
The default entry point. Runs aesop with the \texttt{Pythia} aesop
ruleset, which contains every theorem tagged
\texttt{@[stat\_lemma]}. On aesop failure, falls through to a
short cleanup chain combining \texttt{simp}, \texttt{omega},
\texttt{linarith}, and \texttt{positivity}. The fall-through
ordering is fixed. The lemma database and aesop ruleset are
user-extensible.

\paragraph{\texttt{stats\_ineq}.}
Discharges sub-Gaussian, sub-gamma, Bernstein-Bennett, and
Hoeffding-style inequalities. The \texttt{@[stats\_ineq]} attribute
re-tags a lemma with Mathlib's \texttt{@[bound]}, so the
\texttt{stats\_ineq} tactic dispatches via Mathlib's \texttt{bound}
infrastructure first, then \texttt{positivity}, \texttt{gcongr},
\texttt{linarith}, and \texttt{nlinarith} in that order.
Reconstruction is internal Lean and axiom-clean.

\paragraph{\texttt{prob\_simp}.}
Probability-domain rewrite tactic. The \texttt{@[prob\_simp]}
attribute forwards lemmas to \texttt{@[simp]} and additionally
records the declaration name in a scoped extension for
introspection. The tactic runs \texttt{simp} against the resulting
set with a \texttt{push\_cast} / \texttt{norm\_cast} coercion
fallback for ENNReal / NNReal / Real bridge. The \texttt{pdf\_simp}
syntax is a literal alias. Both forms expand to the same kernel
trace.

\paragraph{\texttt{anytime\_valid}.}
Closes Ville-type maximal-inequality goals for non-negative
supermartingales. The dispatch ladder tries direct
\texttt{ville\_supermartingale} application paths first (countable-time
\texttt{first} with \texttt{exact}/\texttt{refine} variants), then
the finite-horizon and explicit-witness specialisations, and finally
aesop against the named ruleset
\texttt{Pythia.AnytimeValid}. Three syntactic variants are exposed:
the bare form \texttt{anytime\_valid} (countable-time),
\texttt{anytime\_valid (horizon := N)} (finite-horizon, requiring
\texttt{IsProbabilityMeasure} but no integrability), and
\texttt{anytime\_valid using h} (explicit supermartingale witness).
New families register via \texttt{@[anytime\_valid\_lemma]}.

\paragraph{\texttt{z3\_check}.}
External Z3 oracle for linear real arithmetic, with
\texttt{linarith} reconstruction inside Lean. Z3 supplies an
unsat-of-negation verdict. \texttt{linarith} reconstructs the
proof term inside the kernel. The kernel is the final word. If
Z3 says unsat but \texttt{linarith} cannot reconstruct, the tactic
fails loud rather than silently trusting Z3. Z3 cannot smuggle
axioms. This follows the CoqHammer reconstruction pattern
\citep{czajkakaliszyk2018coqhammer}.

\subsection{Registry attributes}
\label{sec:tactic-surface-registries}

The library uses five attribute-driven registries plus a
per-domain typeclass for ad-hoc dispatch:

\begin{itemize}[leftmargin=1.3em,itemsep=0.2em]
\item \texttt{@[stat\_lemma]}: lemma database for the top-level
  \texttt{pythia} dispatcher. Internally a thin wrapper for
  \texttt{@[aesop safe apply (rule\_sets := [Pythia])]}.
\item \texttt{@[stat\_simp]}: curated probability and ENNReal simp
  set. Members are scoped (do not pollute the global \texttt{simp}
  set) and fire only when the \texttt{stat\_simp} tactic is invoked
  or when rung 1 of \texttt{pythia!} dispatches.
\item \texttt{@[stats\_ineq]}: tail-bound and concentration lemmas.
  Forwards to Mathlib's \texttt{@[bound]} attribute and additionally
  records the name for the \texttt{\#stats\_ineqs} introspection
  command.
\item \texttt{@[prob\_simp]}: probability rewrite rules. Forwards to
  \texttt{@[simp]} and records the name for \texttt{\#prob\_simps}.
\item \texttt{@[anytime\_valid\_lemma]}: Ville-style closers and
  admissibility lemmas, populating the
  \texttt{Pythia.AnytimeValid} aesop ruleset.
\item \texttt{DomainCalculator} typeclass: per-domain dispatch
  hook that lets the headline \texttt{pythia} cascade route
  domain-tagged goals through a domain-specific calculator
  instance without touching the central tactic implementation.
\end{itemize}

A separate attribute, \texttt{@[cs\_family]}, is a typed-declaration
tag rather than a tactic registry: it marks a \texttt{CSFamily}
record so it appears in the \texttt{\#cs\_families} catalogue.
User extensions consist of tagging library lemmas with the
appropriate attribute or providing a \texttt{DomainCalculator}
instance. No fork is required.

\subsection{The \texttt{Pythia} aesop ruleset}
\label{sec:tactic-surface-aesop}

The library declares two named aesop rule sets: \texttt{Pythia}
(populated by \texttt{@[stat\_lemma]}) and \texttt{Pythia.AnytimeValid}
(populated by \texttt{@[anytime\_valid\_lemma]}). Direct invocation
is supported via \texttt{aesop (rule\_sets := [Pythia])}, equivalent
to the inner aesop call inside the \texttt{pythia} dispatcher. This
gives users a finer-grained interface when the dispatcher's outer
wrapping is not desired.

\paragraph{Why this shape.}
The two-stage chain mirrors the empirical structure of statistical
proofs in our library. Many goals close on a single registered
lemma application or a short aesop search at stage 1. The remainder
reduce to a linear or arithmetic obligation after rewriting, which
the cleanup chain at stage 2 handles. Routing by goal shape is not
necessary at the top level. The cost of trying stages in order is
small relative to the cost of incorrect early termination.
